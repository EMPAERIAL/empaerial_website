# Context Glossary

## Landing Theme

The visual language used on the public landing-facing pages, including typography, spacing rhythm, surface treatment, and motion tone.

## Admin Theme Parity

Admin pages must use the same visual language as the Landing Theme, while keeping admin-specific information architecture and workflows.

## Admin Structure

The admin page layout and interaction model (tabs, editors, create/manage modes, CRUD flows). This remains admin-specific and is not replaced by landing page section order or marketing layout elements.

## Project Detail Theme Parity

Project detail pages should inherit the same visual language as the Landing Theme and block-based detail pages, while keeping a product-focused information architecture.

## Product Detail Layout

The project detail page is a product page, not an article page. It may use media-heavy sections, technical callouts, specifications, materials, and a final contact block instead of a pure reading flow.

## Project Callouts

Project detail pages may include lightweight per-project callouts anchored to the main product image. In v1, each callout is simple authored content with a label and approximate position, not a complex interactive diagram system.

## Project Detail Content Contract

Every project detail page should include a hero with a main image and optional callouts, a summary, a specifications block, and a final contact block. Gallery, videos, materials, and long-form detail text are optional sections that render only when the project has data.

## Project Hero Priority

The project detail page hero is anchored by the main product image. Title, summary, and key metadata support the hero instead of replacing it as the primary first impression.

## Mobile Callout Fallback

On small screens, project hero callouts should fall back from overlaid annotations to a simple ordered list below the hero image.

## Project Detail Layout Ownership

The project detail page owns the overall section order for its product-page composition. Admin-authored sections provide content for the fixed layout instead of arbitrarily controlling the full page sequence.
